The generator will put transformed SVD data into the `raw` folder. This is not used for anything later on, but can be nice to do migrations or comparisons.
The generator will also put some jsons there with extra info about interrupts and peripheral addresses.

The other folders contain the manually curated yamls of the peripherals.
The exact names are not prescribed. NXP doesn't give us names, so it's up to us to create some sort of consistency there.
